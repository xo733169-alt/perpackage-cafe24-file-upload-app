# Cafe24 고객 UI 안전 수정 기준 문서

- 작성 기준: 2026-07-06
- 대상 영역: Cafe24 고객 화면, 마이쇼핑, 주문조회, 주문상세, 장바구니
- 문서 성격: UI 개선 전 안전 기준 및 조사 문서

## 1. 문서 목적

이 문서는 페르패키지 Cafe24 쇼핑몰의 고객 화면 UI를 나중에 안전하게 개선하기 위한 기준 문서입니다.

마이쇼핑, 주문조회, 주문상세, 장바구니 영역은 주문, 결제, 배송, 취소, 교환, 반품, 회원 기능과 연결되어 있습니다. 따라서 화면이 불편해 보여도 바로 구조를 바꾸기보다, 어떤 부분은 수정해도 되는지와 어떤 부분은 건드리면 안 되는지를 먼저 구분해야 합니다.

이 문서의 목적은 다음과 같습니다.

- 마이쇼핑, 주문조회, 주문상세, 장바구니 UI 개선 전 안전 기준을 정리합니다.
- Cafe24 스킨 코드에서 건드려도 되는 영역과 건드리면 안 되는 영역을 구분합니다.
- 주문, 결제, 배송, 취소, 교환, 반품, 회원 기능이 깨지지 않도록 합니다.
- 디자인 개선은 기능 안정성을 확인한 뒤 작은 영역부터 단계적으로 진행합니다.

## 3. 건드리면 안 되는 코드 기준

아래 항목은 “주의 또는 수정 금지” 기준으로 봅니다. 디자인 작업 중 실수로 바꾸면 주문, 결제, 배송, 회원 기능이 깨질 수 있습니다.

| 항목 | 예시 | 조심해야 하는 이유 |
|---|---|---|
| `module` 속성 | `module="Myshop_OrderHistoryDetailBasic"` | Cafe24가 데이터를 불러오고 반복 출력하는 기준입니다. 바꾸면 주문 상품, 옵션, 버튼이 사라질 수 있습니다. |
| Cafe24 템플릿 변수 | `{$order.order_id}`, `{$option_str}`, `{$status}`, `{$param_list}` | 실제 주문번호, 옵션, 상태값이 표시되는 자리입니다. 지우거나 이름을 바꾸면 고객에게 필요한 값이 표시되지 않습니다. |
| display 제어 클래스 | `{$option_display|display}`, `{$button_display|display}`, `{$cancel_display|display}` | Cafe24가 조건에 따라 보이거나 숨겨야 하는 영역을 제어합니다. 잘못 바꾸면 필요 없는 버튼이 보이거나 필요한 버튼이 숨겨질 수 있습니다. |
| `onclick` 동작 | `{$action_withdraw_cancel}`, `{$action_purchase_confirmation}`, `{$shipping_trace_popup}` | 주문취소, 구매확정, 배송조회 같은 실제 동작과 연결됩니다. 수정하면 버튼은 보여도 작동하지 않을 수 있습니다. |
| 주문/결제/배송/회원 관련 버튼 | 주문취소, 구매확정, 배송지 변경, 교환철회, 반품철회, 취소철회, 현금영수증, 세금계산서 | 고객의 실제 주문 처리와 연결된 버튼입니다. 위치 조정도 신중해야 하고, 기능 코드는 건드리지 않습니다. |
| 장바구니 수량/옵션/가격 계산 영역 | 수량 변경, 옵션 변경, 상품금액, 배송비, 할인, 쿠폰, 적립금 | 금액 계산과 주문 흐름에 직접 영향을 줍니다. 디자인 변경 중 입력값 이름이나 버튼 동작을 바꾸면 주문 오류가 날 수 있습니다. |
| 회원정보/배송주소록 영역 | 회원정보 수정, 주소 추가/수정/삭제 | 고객 개인정보 저장과 배송지 선택에 연결됩니다. 입력 필드와 저장 버튼 구조를 바꾸면 저장이 실패할 수 있습니다. |
| 취소/교환/반품 신청 기능 | 신청, 철회, 사유 선택, 환불 정보 | 주문 처리와 고객 응대에 직접 연결됩니다. 문구 수정도 고객 오해가 생기지 않게 확인해야 합니다. |
| Cafe24 기본 모듈 반복 구조 | 상품 목록, 주문 목록, 옵션 목록 반복 블록 | 반복 블록을 잘못 삭제하면 일부 주문이나 상품만 보이거나 전체 목록이 깨질 수 있습니다. |

## 4. 비교적 안전하게 수정 가능한 영역

아래 항목은 비교적 안전한 UI 개선 후보입니다. 그래도 PC와 모바일 화면에서 확인한 뒤 적용합니다.

- 안내 문구 추가
- 버튼 문구 변경, 단 기능 버튼의 의미를 바꾸지 않는 범위
- 여백, 테두리, 배경색, 폰트 크기 조정
- 카드형 레이아웃 정리
- 고객센터 안내 영역 정리
- 사용하지 않는 메뉴 숨김, 단 삭제 전 확인 필요
- 새 안내 박스 추가
- 외부 링크 버튼 추가, 단 주문/결제 기능과 분리된 위치
- 아이콘 또는 배지 추가
- FAQ/도움말 안내 추가

### 숨김과 삭제의 차이

숨김 처리와 삭제는 다릅니다.

- 숨김: 화면에서는 보이지 않지만, 원본 코드를 되돌릴 수 있습니다.
- 삭제: 코드를 완전히 제거하므로 복구가 어렵고, 숨겨진 기능이 함께 사라질 수 있습니다.

처음에는 삭제보다 `display:none` 방식, 주석 처리, 원본 백업 기준으로 접근하는 것이 안전합니다. 단, Cafe24 모듈 내부에서는 숨김 처리도 조건부 표시와 충돌할 수 있으므로 적용 후 반드시 기능 테스트를 합니다.

## 5. 화면별 UI 개선 방향 초안

### 마이쇼핑 메인

- 현재 기능 버튼이 많고 정보가 흩어져 있을 수 있습니다.
- 실제 사용하는 메뉴와 사용하지 않는 메뉴를 먼저 구분합니다.
- 주문조회, 파일 상태조회, 상담문의, 배송주소록 등 핵심 메뉴 중심으로 정리할 수 있습니다.
- 회원정보, 쿠폰, 적립금처럼 고객 계정과 연결된 기능은 숨기기 전 실제 사용 여부를 확인합니다.

### 주문조회 목록

- 주문번호, 상품명, 주문상태, 업로드 파일 ID가 잘 보이는지 확인합니다.
- 1차에서는 큰 안내 박스를 주문마다 반복하지 않습니다.
- 나중에 목록 상단에 공통 안내문이나 파일 상태조회 바로가기를 넣는 방식을 검토할 수 있습니다.
- 주문상세 이동 링크와 주문 상태 표시는 유지합니다.

### 주문상세

- 현재 파일 상태조회 안내 박스 적용이 완료된 화면입니다.
- 업로드 파일 ID 근처에 안내를 두는 방향을 유지합니다.
- 나중에 자동 입력 방식은 별도 검토합니다.
- 주문취소, 구매확정, 배송조회, 교환/반품 버튼은 기능 코드와 위치를 신중하게 다룹니다.

### 장바구니

- 주문 완료 전 단계이므로 파일 상태조회 안내 위치로는 부적합합니다.
- 장바구니에서는 업로드 여부나 옵션 확인 정도만 다루는 것이 안전합니다.
- 수량, 옵션, 가격 계산 영역은 수정에 특히 주의합니다.
- 결제 버튼과 외부 간편결제 버튼은 숨기거나 위치를 바꾸기 전 반드시 확인합니다.

### 주문완료

- “주문 후 파일 상태는 마이쇼핑 주문상세에서 확인 가능” 정도의 안내는 검토할 수 있습니다.
- 주문 직후에는 Webhook 자동 연결 전일 수 있으므로 “확정”, “완료”, “제작 시작”처럼 단정적인 표현은 피합니다.
- 고객이 주문번호를 확인할 수 있는 위치와 함께 안내하는 것이 좋습니다.

## 6. Cafe24 고객 UI 개선 작업 순서 제안

1. 현재 화면 캡처를 수집합니다.
2. 실제 사용하는 메뉴와 사용하지 않는 메뉴를 구분합니다.
3. PC/모바일 스킨 분리 여부를 확인합니다.
4. 건드리면 안 되는 `module`, 변수, `onclick` 위치를 표시합니다.
5. 작은 안내 영역부터 수정합니다.
6. PC 화면을 확인합니다.
7. 모바일 화면을 확인합니다.
8. 주문, 취소, 구매확정, 배송조회 등 핵심 버튼을 확인합니다.
9. 문제가 없으면 다음 영역으로 진행합니다.

## 9. 디자인 개선 원칙

페르패키지 고객 UI는 기능이 먼저 안정적으로 동작해야 합니다. 디자인 개선은 아래 원칙을 기준으로 진행합니다.

- 흰색/남색 중심의 깔끔한 화면을 유지합니다.
- 안내 문구는 짧고 명확하게 씁니다.
- 고객이 눌러야 하는 버튼만 강조합니다.
- 결제, 취소, 반품 같은 기능 버튼은 기존 위치를 크게 바꾸지 않습니다.
- 한 번에 전체를 바꾸지 않고 작은 영역부터 수정합니다.
- PC와 모바일을 함께 확인합니다.
- 고객이 주문번호와 업로드 파일 ID를 쉽게 찾을 수 있게 안내합니다.
- “제작 확정”, “인쇄 확정”, “납기 확정”처럼 운영상 오해를 줄 수 있는 표현은 피합니다.

## 10. 보안 기준

고객 화면, 문서, 테스트 결과, 보고서에는 아래 정보가 원문으로 노출되지 않게 합니다.

- `storage_path` 노출 금지
- signed URL 원문 노출 금지
- `token_hash` 또는 raw token 노출 금지
- Webhook raw payload 전체 노출 금지
- Cafe24 Admin API 응답 전체 노출 금지
- 고객 다운로드 버튼은 기본 제공하지 않음
- `file_id` 단독 조회 유도 금지
- 주문번호와 업로드 파일 ID 조합 조회 기준 유지
- 재업로드 링크는 별도 안내받은 링크만 사용
- service role key, Naver Object Storage key, OAuth token 노출 금지
