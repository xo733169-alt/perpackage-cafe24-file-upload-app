# 재업로드 요청 사유 프리셋 버튼 추가 보고서

작성일: 2026-07-01

프로젝트: `perpackage-cafe24-file-upload-app`

## 1. 작업 목적

`/admin?file_id=<file_id>` 파일 상세 화면의 `재업로드 요청 안내문` 기능에 자주 사용하는 재업로드 요청 사유 프리셋 버튼을 추가했다.

관리자가 매번 사유를 직접 입력하지 않아도 버튼을 눌러 사유 입력칸에 표준 문구를 넣고, 고객 안내문에 즉시 반영되도록 하는 것이 목적이다.

## 2. 수정한 파일 목록

```txt
src/components/ReuploadRequestMessagePanel.tsx
docs/gpt-report-reupload-reason-presets-20260701.md
```

## 3. 주요 변경 내용

- `ReuploadRequestMessagePanel`에 자주 쓰는 사유 프리셋 목록 추가
- 재업로드 요청 사유 textarea 위에 `자주 쓰는 사유` 버튼 영역 추가
- 프리셋 버튼 클릭 시 기존 사유 입력값을 선택한 문구로 교체
- 사유가 바뀌면 기존 안내문 생성 로직을 통해 고객 안내문에 즉시 반영
- 복사 성공/실패 피드백은 사유 변경 시 초기화

## 4. 추가한 프리셋 목록

| 버튼 라벨 | 입력 문구 |
| --- | --- |
| 폰트 아웃라인 필요 | 폰트 아웃라인 처리가 필요합니다. |
| 칼선 누락 | 칼선 파일이 누락되어 확인이 필요합니다. |
| 이미지 해상도 확인 필요 | 이미지 해상도 확인이 필요합니다. |
| 파일 열림 오류 | 첨부 파일이 정상적으로 열리지 않아 재업로드가 필요합니다. |
| 인쇄용 PDF/AI 파일 필요 | 인쇄용 PDF 또는 AI 파일로 다시 전달이 필요합니다. |
| 전개도 위치 확인 필요 | 박스 전개도와 디자인 위치 확인이 필요합니다. |

## 5. 프리셋 클릭 후 안내문 반영 방식

프리셋 버튼을 클릭하면 `reason` 상태값이 해당 프리셋 문구로 교체된다.

기존 안내문 생성 함수는 `reason` 값이 있을 때 아래 줄을 안내문에 포함한다.

```txt
확인 필요 사항: {선택한 프리셋 문구}
```

사유 입력값이 비어 있으면 기존처럼 `확인 필요 사항` 줄은 안내문에 표시하지 않는다.

- authorization
- bearer
- token
- client secret
- secret
- signature

- x-api-key
- Supabase service role key
- Naver Object Storage access key
- Naver Object Storage secret key

- Webhook raw payload 전체

## 13. 운영자가 /admin에서 확인해야 할 테스트 순서

1. `/admin` 로그인
2. `file_id` 검색
3. 파일 상세 화면의 `재업로드 요청 안내문` 섹션 확인
4. `자주 쓰는 사유` 버튼 6개가 표시되는지 확인
5. `폰트 아웃라인 필요` 버튼 클릭
6. 재업로드 요청 사유 입력칸에 문구가 들어가는지 확인
7. 고객 안내문에 `확인 필요 사항: 폰트 아웃라인 처리가 필요합니다.`가 반영되는지 확인
8. 다른 프리셋 버튼을 눌렀을 때 기존 입력값이 새 문구로 교체되는지 확인
9. 직접 사유 입력도 기존처럼 동작하는지 확인
10. 안내문 복사 버튼과 복사 성공 피드백이 유지되는지 확인
11. 파일 상태 변경, 상태 변경 이력, 주문번호 연결 이력, 다운로드 버튼이 기존처럼 유지되는지 확인
