# Cafe24 주문상세 파일 상태조회 안내 박스 적용 완료 기록

- 작성 기준: 2026-07-06
- 대상 화면: Cafe24 고객 마이페이지 주문조회 > 주문상세 화면
- 문서 성격: 운영 반영 완료 기록

## 1. 적용 목적

Cafe24 고객 마이페이지 주문조회 > 주문상세 화면에서 고객이 주문번호와 업로드 파일 ID를 확인한 뒤, 파일 상태조회 페이지로 이동할 수 있도록 안내 박스를 적용했습니다.

운영 방향은 다음과 같습니다.

- 고객은 `/file-status`에서 주문번호와 업로드 파일 ID를 직접 입력합니다.
- 파일 다운로드는 고객 화면에서 제공하지 않습니다.
- 상품상세와 장바구니가 아니라, 주문 완료 후 다시 확인하는 주문상세 화면에 안내를 둡니다.
- URL에 `order_id`나 `file_id`를 자동으로 붙이지 않습니다.
- `file_id` 자동 파싱 JavaScript는 사용하지 않습니다.

## 2. 실제 적용 위치

적용 위치는 다음과 같습니다.

- Cafe24 고객 마이페이지 주문조회 > 주문상세 화면
- 주문상품 영역
- 업로드 파일 ID가 표시되는 옵션 영역 아래
- 주문처리상태 영역인 `prdFoot` 위

적용하지 않은 위치는 다음과 같습니다.

- 상품상세 화면
- 장바구니 화면
- 주문조회 목록의 각 주문 행 내부

## 3. 실제 적용 방식

적용 방식은 다음과 같습니다.

- 짧은 문구 버전 안내 박스 적용
- 버튼 문구: `업로드 파일 상태 조회하기`
- 버튼 링크: `https://perpackage-cafe24-file-upload-app.vercel.app/file-status`
- 버튼은 새 창 또는 새 탭으로 열리도록 적용
- URL 파라미터 자동 입력 없음
- `file_id` 자동 파싱 JavaScript 없음
- 고객이 주문번호와 업로드 파일 ID를 직접 입력하는 1차 방식 유지

## 6. 현재 남은 개선 후보

현재 기능 문제는 아니지만, 추후 아래 개선을 검토할 수 있습니다.

- 모바일에서 긴 UUID 형식의 업로드 파일 ID 입력칸 가독성 개선
- `/file-status` 입력칸의 긴 텍스트 표시 방식 개선
- 마이페이지 메인 또는 주문조회 목록 상단에 짧은 공통 안내 추가
- URL 자동 입력 방식은 2차 개선 후보로 유지
- 조회 시도 제한 또는 rate limit 검토

## 7. 되돌리기 참고

운영 스킨에서 안내 박스를 되돌려야 할 경우 아래 기준으로 처리합니다.

- Cafe24 주문상세 템플릿에서 `perpackage-file-status-box` 영역 제거
