# Cafe24 마이쇼핑 메인 UI 정리 1차 설계 문서

- 작성 기준: 2026-07-06
- 대상 화면: Cafe24 고객 마이쇼핑 메인
- 문서 성격: UI 개선 전 1차 설계 문서

## 1. 문서 목적

이 문서는 Cafe24 마이쇼핑 메인 화면을 고객이 더 쉽게 사용할 수 있도록 정리하기 위한 1차 설계 문서입니다.

실제 Cafe24 운영 스킨을 수정하기 전에 핵심 메뉴, 보조 메뉴, 숨김 후보를 구분하고, 주문, 회원, 배송, 상담, 파일 상태조회 흐름을 고객이 쉽게 찾도록 하는 것이 목적입니다.

이번 설계에서는 메뉴를 바로 삭제하지 않고, 숨김 후보를 따로 관리하는 방식으로 접근합니다.

## 2. 현재 마이쇼핑 문제점 요약

- 메뉴가 많고 고객이 무엇을 눌러야 할지 헷갈릴 수 있습니다.
- 좌측 FAQ/고객센터 메뉴에 오래된 스킨 샘플성 메뉴가 섞여 있습니다.
- 실제 운영하지 않는 메뉴가 고객에게 노출될 가능성이 있습니다.
- 파일 상태조회 기능은 생겼지만, 마이쇼핑 메인에서는 아직 바로 찾기 어려울 수 있습니다.
- PC와 모바일 모두 메뉴 배치와 가독성 정리가 필요합니다.
- 주문, 회원, 배송, 상담처럼 중요한 기능과 보조 기능이 같은 무게로 보일 수 있습니다.

## 5. 숨김 후보 메뉴

아래 메뉴는 삭제 대상이 아니라 숨김 검토 대상입니다. 실제 사용 여부를 확인한 뒤 숨김 처리 여부를 결정합니다.

- 좋아요
- 대량구매문의 관리, 실제 사용하지 않는 경우
- 직접추가가능
- 상품입고일정
- 이벤트/제휴/기타
- 이달의 베스트 후기
- 해외배송문의, 해외배송 미운영 시
- 오프라인 매장안내, 최신 안내가 없다면

숨김 후보를 처리할 때도 원본 코드를 백업하고, PC/모바일에서 표시와 기능을 확인한 뒤 진행합니다.

## 6. 추천 레이아웃 구조

### 상단

- 고객 이름 또는 고객 등급 표시
- 주문 처리 현황 요약
- 최근 주문 또는 주문상태 요약

### 핵심 바로가기 카드

1차 핵심 카드로 아래 메뉴를 위쪽에 배치합니다.

- 주문조회
- 파일 상태 조회
- 회원정보 수정
- 배송 주소록
- 카카오톡 상담
- 네이버톡톡 상담

### 보조 메뉴

핵심 메뉴 아래 또는 하단 영역에 배치합니다.

- 쿠폰
- 적립금
- 게시물관리
- 관심상품
- 최근본 상품

### 고객센터

상담 채널과 FAQ를 분리해 정리합니다.

- FAQ
- 주문/결제/영수증
- 배송/상품
- 취소/반품/교환

### 숨김 후보

숨김 후보 메뉴는 하단 접힘 영역 또는 미노출 대상으로 검토합니다.

- 직접추가가능
- 상품입고일정
- 이벤트/제휴/기타
- 이달의 베스트 후기
- 운영하지 않는 해외배송/대량구매/좋아요 관련 메뉴

## 7. 파일 상태조회 카드 추가 여부

현재 Cafe24 고객 마이페이지 주문조회 > 주문상세 화면에는 파일 상태조회 안내 박스가 적용되어 있습니다.

마이쇼핑 메인에 파일 상태조회 카드를 추가하면 고객이 `/file-status` 페이지를 더 쉽게 찾을 수 있습니다.

다만 마이쇼핑 메인에는 주문번호와 업로드 파일 ID가 바로 보이지 않으므로, 고객은 주문상세에서 값을 확인해야 할 수 있습니다.

1차 UI 개선에서는 카드 추가를 검토하되, 아래 문구 기준을 함께 적용하는 것이 안전합니다.

- “주문번호와 업로드 파일 ID가 필요합니다.”
- “업로드 파일 ID는 주문상세의 상품 옵션 영역에서 확인할 수 있습니다.”
- 링크는 `/file-status`로 이동합니다.
- URL 자동 입력은 하지 않습니다.
- 고객 다운로드 버튼은 제공하지 않습니다.

## 8. 좌측 고객센터 영역 정리 방향

- 상담 채널과 FAQ를 분리합니다.
- 상담 채널은 네이버톡톡, 카카오톡채널, 폼메일 상담만 남길지 검토합니다.
- FAQ는 주문/결제/영수증, 배송/상품, 취소/반품/교환처럼 실제 운영 내용이 있는 항목 중심으로 정리합니다.
- 직접추가가능, 상품입고일정, 이벤트/제휴/기타, 이달의 베스트 후기 등은 숨김 후보로 둡니다.
- 상품문의하기/상품문의, 주문조회/주문배송조회처럼 중복 가능성이 있는 메뉴는 하나로 통합할지 검토합니다.
- 주문/회원/배송 관련 기능 링크는 유지합니다.

## 9. PC/모바일 설계 기준

### PC

- 카드형 메뉴를 2~3열로 배치합니다.
- 핵심 메뉴를 위쪽에 배치합니다.
- 고객센터는 좌측 또는 하단에 정리합니다.
- 보조 메뉴는 핵심 메뉴보다 낮은 시각적 우선순위로 둡니다.
- 숨김 후보 메뉴는 접힘 영역 또는 미노출로 검토합니다.

### 모바일

- 핵심 메뉴를 1열 카드로 표시합니다.
- 버튼 크기를 충분히 크게 둡니다.
- 숨김 후보 메뉴는 접힘 영역 또는 미노출로 둡니다.
- 파일 상태조회 버튼은 너무 아래로 내려가지 않게 배치합니다.
- 상담 채널 버튼은 한눈에 구분되도록 짧은 문구를 사용합니다.

## 10. 안전 수정 기준

- `module` 속성은 수정하지 않습니다.
- `{$변수}` 형태의 Cafe24 템플릿 변수는 수정하지 않습니다.
- `onclick` 동작은 수정하지 않습니다.
- 주문조회, 회원정보, 배송주소록 기능은 삭제하지 않습니다.
- 숨김 전 원본 코드를 백업합니다.
- 삭제보다 숨김을 우선합니다.
- 한 번에 전체 수정하지 말고 메뉴 그룹별로 적용합니다.
- PC와 모바일 확인은 필수입니다.
- 주문, 결제, 취소, 반품, 교환 관련 링크는 기능 테스트 후 조정합니다.
