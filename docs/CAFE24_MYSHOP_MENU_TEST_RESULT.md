- 마이쇼핑 UI 개선 전에 실제 메뉴가 정상 이동되는지 확인한 결과를 기록합니다.
- 정상 메뉴, 빈 화면 메뉴, 사용 여부 확인 메뉴, 숨김 후보 메뉴를 구분합니다.
- 메뉴는 바로 삭제하지 않고 숨김 후보로 관리합니다.

## 3. 1차 유지/숨김 후보 정리

### 반드시 유지

- 주문조회
- 회원정보
- 배송 주소록 관리

위 메뉴는 고객 주문, 계정, 배송 기능과 직접 연결되므로 삭제하지 않습니다. 위치와 문구는 개선할 수 있지만 기능 자체는 유지합니다.

### 유지 가능

- 게시물관리
- 적립금
- 쿠폰

게시물관리는 실제 게시글이 확인되었으므로 고객 문의/게시판 운영과 연결된다면 유지합니다. 적립금과 쿠폰은 정상 표시되지만 현재 값이 0원이거나 0장이므로, 실제 운영 정책에 따라 유지 여부를 결정합니다.

### 사용 여부 확인 필요

- 관심상품
- 최근본 상품
- 좋아요
- 대량구매 문의 관리

위 메뉴는 기능적으로 이동되지만, 페르패키지 쇼핑몰 운영에서 실제 필요한지 확인해야 합니다.

### 숨김 후보

- 좋아요: 운영 미사용 시
- 대량구매 문의 관리: 운영 미사용 시
- 쿠폰/적립금: 정책 미운영 시
- 관심상품/최근본 상품: 쇼핑몰형 UX에서 필요 없다고 판단될 경우

숨김 후보는 삭제 대상이 아닙니다. 실제 사용 여부와 링크 동작을 확인한 뒤 숨김 여부를 결정합니다.

## 4. 일반 고객 계정으로 추가 확인할 항목

- [ ] 회원정보 수정이 일반 고객 계정에서 정상 작동하는지 확인
- [ ] 쿠폰/적립금이 일반 고객 화면에서도 정상 표시되는지 확인
- [ ] 관심상품/최근본 상품이 일반 고객 사용 흐름에서 필요한지 확인
- [ ] 좋아요 기능을 실제 운영할지 확인
- [ ] 대량구매 문의 관리 기능을 실제 사용할지 확인

## 5. UI 개선 방향 초안

- 1차 UI 개선은 메뉴 삭제가 아니라 정렬, 강조, 숨김 후보 분리부터 진행합니다.
- 주문조회, 파일 상태조회, 상담문의, 배송주소록을 핵심 메뉴로 묶는 방향을 검토합니다.
- 관심상품, 최근본 상품, 좋아요는 쇼핑몰형 UX 유지 여부에 따라 위치를 낮출 수 있습니다.
- 적립금과 쿠폰은 실제 운영 정책에 따라 유지 여부를 결정합니다.
- 대량구매 문의 관리는 실제 사용하지 않으면 숨김 후보로 둡니다.
- 회원정보 수정은 관리자 계정 제한이 있으므로 일반 고객 계정 테스트 후 판단합니다.

## 6. 안전 기준

- 메뉴를 바로 삭제하지 않습니다.
- 숨김 전 링크와 기능을 확인합니다.
- `module` 속성은 수정하지 않습니다.
- `{$변수}` 형태의 Cafe24 템플릿 변수는 수정하지 않습니다.
- `onclick` 동작은 수정하지 않습니다.
- 고객 주문, 회원, 배송 기능은 삭제하지 않습니다.
- PC와 모바일을 모두 확인합니다.
- 수정 전 Cafe24 스킨 원본을 백업합니다.
