# Cafe24 업로드 위젯 candidate 최종 점검표

작성일: 2026-07-09

이 문서는 `public/cafe24/product-upload-widget-candidate.js`를 운영 `public/cafe24/product-upload-widget.js`로 승격하기 전에 확인할 기능, 테스트 항목, 주의사항을 정리한 최종 점검표입니다.

운영 위젯 승격 전까지 Cafe24 테스트 스킨에서는 candidate JS를 사용하고, 운영 상품상세에는 기존 운영 JS를 유지합니다.

## 1. 현재 candidate 주요 기능

- 옵션 선택 전에는 업로드 카드 전체를 숨깁니다.
- 옵션 선택 후에는 업로드 카드 전체를 표시합니다.
- 선택상품 삭제 후에는 업로드 카드 전체를 다시 숨깁니다.
- 선택상품 row는 최신 1개만 유지합니다.
- 선택상품 수량은 1개로 고정합니다.
- 파일은 1개 업로드 기준입니다.
- 여러 파일은 ZIP 1개로 압축하도록 안내합니다.
- 장바구니 담기 성공 팝업 확인 후 상품상세를 새로고침합니다.
- 새 파일을 업로드하면 기존 file_id를 새 file_id로 교체합니다.
- file_id는 Cafe24 추가 입력 옵션 중 `업로드 파일 ID` 입력칸에만 자동 입력합니다.
- 후가공, 컬러지 등 다른 추가 옵션에는 file_id가 들어가지 않도록 방지합니다.
- 상품상세의 `업로드 파일 ID` row는 DOM과 input 값을 유지하되 고객 화면에서는 시각적으로 숨깁니다.
- 장바구니, 주문서, 주문 완료, 마이쇼핑 고객 화면에서는 별도 JS인 `public/cafe24/customer-hide-upload-file-id-option.js`로 업로드 파일 ID 표시를 숨깁니다.
- JPG, JPEG, PNG 낱장 업로드를 허용합니다.
- 서버 업로드 보안 정책을 반영합니다.
  - 직접 업로드 허용: AI, PDF, EPS, JPG, JPEG, PNG, ZIP
  - ZIP 내부 허용: AI, PDF, EPS, JPG, JPEG, PNG
  - 위험 확장자 차단
  - ZIP 내부 위험 파일, 안전하지 않은 경로, 손상 ZIP, 비밀번호 ZIP 차단
- 고객 안내 문구는 최대 100MB 기준입니다.
- 서버에서 반환한 한국어 오류 메시지를 고객 화면에 표시합니다.

## 2. 최종 테스트 체크리스트

### 상품상세

- [ ] 옵션 선택 전 업로드 카드가 숨겨져 있다.
- [ ] 옵션 선택 후 업로드 카드가 표시된다.
- [ ] 업로드 파일 ID 입력 row가 고객 화면에 노출되지 않는다.
- [ ] JPG 업로드가 성공한다.
- [ ] PNG 업로드가 성공한다.
- [ ] PDF 업로드가 성공한다.
- [ ] AI 업로드가 성공 가능하다.
- [ ] EPS 업로드가 성공 가능하다.
- [ ] CSV 업로드가 차단된다.
- [ ] EXE 업로드가 차단된다.
- [ ] 위험 파일이 포함된 ZIP 업로드가 차단된다.
- [ ] 허용 파일만 포함된 ZIP 업로드가 성공한다.
- [ ] 새 파일 업로드 시 업로드 파일 ID 입력칸의 file_id가 새 값으로 교체된다.
- [ ] 후가공 입력칸에 file_id가 들어가지 않는다.
- [ ] 컬러지 입력칸에 file_id가 들어가지 않는다.
- [ ] 파일 없이 장바구니 담기가 가능하다.
- [ ] 파일 없이 구매하기가 가능하다.
- [ ] 업로드 중에는 장바구니 담기가 잠깐 차단된다.
- [ ] 업로드 중에는 구매하기가 잠깐 차단된다.

### 장바구니

- [ ] 장바구니 담기 성공 후 상품상세가 새로고침된다.
- [ ] 장바구니 고객 화면에서 업로드 파일 ID가 보이지 않는다.
- [ ] 지류, 사이즈, 수량, 인쇄추가, 후가공, 컬러지 등 다른 옵션은 정상 표시된다.
- [ ] 수량이 1개로 유지된다.

### 주문서 / 결제

- [ ] 주문서 고객 화면에서 업로드 파일 ID가 보이지 않는다.
- [ ] 다른 옵션은 정상 표시된다.
- [ ] 결제 진행이 가능하다.

### 주문 완료 / 관리자

- [ ] Cafe24 관리자 주문상세에는 업로드 파일 ID가 존재한다.
- [ ] Webhook 로그가 `auto_linked` 또는 `already_linked`로 처리된다.
- [ ] `/admin`에서 주문번호 검색이 가능하다.
- [ ] `/admin`에서 file_id 검색이 가능하다.
- [ ] 최근 업로드 파일 목록에 주문번호 연결 상태가 표시된다.

## 3. 운영 승격 전 주의사항

- 운영 `public/cafe24/product-upload-widget.js`는 아직 미수정 상태로 유지합니다.
- 운영 승격 시 candidate와 운영 파일의 diff를 먼저 확인합니다.
- Webhook 로직은 수정하지 않습니다.
- `/admin`, `/reupload`, `/file-status`는 수정하지 않습니다.
- Cafe24 운영 스킨 HTML은 직접 수정하지 않습니다.
- 운영 반영 직후 상품상세가 흰 화면 없이 정상 렌더링되는지 확인합니다.
- 문제가 발생하면 기존 운영 JS로 즉시 롤백할 수 있어야 합니다.
- 운영 승격 후에도 file_id 원문을 console에 출력하지 않습니다.
- signed URL, storage_path, token, secret, raw payload 등 민감정보가 고객 화면이나 로그에 노출되지 않는지 확인합니다.

## 4. 운영 승격 판단 기준

아래 조건을 모두 만족하면 운영 `product-upload-widget.js` 승격을 검토할 수 있습니다.

- [ ] 테스트 스킨에서 상품상세 업로드 흐름이 정상이다.
- [ ] JPG 업로드가 정상이다.
- [ ] PNG 업로드가 정상이다.
- [ ] PDF 업로드가 정상이다.
- [ ] CSV 업로드 차단이 정상이다.
- [ ] EXE 업로드 차단이 정상이다.
- [ ] 장바구니 고객 화면에서 file_id가 보이지 않는다.
- [ ] 주문서 고객 화면에서 file_id가 보이지 않는다.
- [ ] Cafe24 관리자 주문상세에는 file_id가 존재한다.
- [ ] Webhook `auto_linked` 또는 `already_linked`를 확인했다.
- [ ] 모바일 상품상세에서 업로드 카드 표시, 파일 선택, 업로드가 정상이다.
- [ ] 모바일 장바구니에서 file_id가 보이지 않는다.
- [ ] 모바일 주문서에서 file_id가 보이지 않는다.
- [ ] Vercel Production 환경변수 `UPLOAD_MAX_FILE_SIZE_MB=100` 적용을 확인했다.
- [ ] 100MB 이하 파일 업로드가 허용되고, 100MB 초과 파일은 차단되는지 확인했다.

## 5. 남은 후속 작업

- 고객 주문조회 화면에서 파일 상태를 표시합니다.
- 고객 주문조회 화면에서 재업로드 버튼을 연동합니다.
- 운영 매뉴얼을 최신화합니다.
- 직원용 짧은 실무 체크리스트를 작성합니다.

## 6. 운영 승격 시 권장 순서

1. 테스트 스킨에서 최종 체크리스트를 한 번 더 수행합니다.
2. candidate와 운영 위젯 diff를 확인합니다.
3. 운영 위젯에 필요한 변경만 반영합니다.
4. `node --check public/cafe24/product-upload-widget.js`를 실행합니다.
5. `node node_modules/typescript/bin/tsc --noEmit`을 실행합니다.
6. `node node_modules/next/dist/bin/next build`를 실행합니다.
7. main에 커밋 및 push합니다.
8. Vercel Production 배포 완료를 확인합니다.
9. Cafe24 운영 상품상세에서 흰 화면 여부를 먼저 확인합니다.
10. 상품상세, 장바구니, 주문서, 주문 완료, 관리자, Webhook 로그를 순서대로 확인합니다.

## 7. 롤백 기준

아래 문제가 발생하면 운영 승격 코드를 유지하지 않고 즉시 롤백합니다.

- 상품상세가 흰 화면으로 멈춘다.
- 옵션 선택 또는 삭제 후 Cafe24 기본 구매 흐름이 깨진다.
- 파일 없이 장바구니 또는 구매가 막힌다.
- 업로드 성공 후 Cafe24 관리자 주문상세에 file_id가 전달되지 않는다.
- Webhook 로그가 반복적으로 `no_file_id`로 처리된다.
- 장바구니 또는 주문서에서 file_id 숨김 때문에 다른 옵션이 같이 사라진다.
