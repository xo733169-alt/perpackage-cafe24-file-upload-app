# Cafe24 고객 주문상세 파일 상태 조회 안내 박스 적용 가이드

- 작성 기준: 2026-07-06
- 대상 화면: Cafe24 고객 마이페이지 주문조회 > 주문상세 화면
- 문서 성격: 운영 스킨 적용 전 위치 확인 가이드

## 1. 문서 목적

이 문서는 Cafe24 고객 마이페이지 주문조회 > 주문상세 화면에 “업로드 파일 상태 조회 안내 박스”를 넣기 전, 실제 적용 위치와 운영 기준을 확인하기 위한 가이드입니다.

고객은 주문상세 화면에서 주문번호와 업로드 파일 ID를 확인한 뒤 `/file-status` 페이지로 이동해 직접 상태를 조회합니다.

1차 적용 방향은 다음과 같습니다.

- 상품상세에는 파일 업로드 위젯만 둡니다.
- 장바구니에는 파일 상태조회 안내를 넣지 않습니다.
- 파일 상태조회 안내는 주문 완료 후 고객이 다시 확인하는 마이페이지 주문조회 > 주문상세 화면에 둡니다.
- 주문조회 목록 화면은 2차 개선 후보로 두고, 1차에서는 반복 안내 박스를 넣지 않습니다.
- `/file-status` URL에 `order_id`나 `file_id`를 자동으로 붙이지 않습니다.
- 고객은 `/file-status`에서 주문번호와 업로드 파일 ID를 직접 입력합니다.
- 고객 화면에는 파일 다운로드 버튼을 제공하지 않습니다.

## 2. 현재 화면 기준 위치 판단

### 주문조회 목록 화면

Cafe24 고객 마이페이지 주문조회 목록 화면(`/myshop/order/list.html`)에서는 주문번호와 업로드 파일 ID가 보일 수 있습니다.

다만 주문 목록은 여러 주문이 반복해서 표시되는 화면이므로, 각 주문마다 안내 박스를 넣으면 화면이 복잡해질 수 있습니다. 1차 적용에서는 주문조회 목록에 큰 안내 박스를 넣지 않고, 필요하면 2차에서 상단 공통 안내문 정도만 검토합니다.

### 주문조회 > 주문상세 화면

Cafe24 고객 마이페이지 주문조회 > 주문상세 화면(`/myshop/order/detail.html?order_id=...`)은 1차 적용 위치로 가장 적합합니다.

이 화면에서는 고객이 아래 정보를 함께 확인할 수 있습니다.

- 주문번호
- 주문 상품 정보
- 상품 옵션 또는 추가 입력 옵션에 표시된 업로드 파일 ID

고객이 주문번호와 업로드 파일 ID를 같은 화면에서 확인한 뒤 `/file-status`로 이동할 수 있으므로, 상태 조회 안내 박스는 주문상세 화면에 배치하는 것을 1차 기준으로 합니다.

## 3. 권장 삽입 위치

### 1순위

주문상세 화면의 주문 상품 정보 영역 바로 아래, 또는 업로드 파일 ID가 보이는 상품 옵션 영역 바로 아래에 넣는 것을 권장합니다.

이 위치가 적합한 이유는 다음과 같습니다.

- 고객이 주문번호와 업로드 파일 ID를 확인한 직후 상태 조회 안내를 볼 수 있습니다.
- 결제/배송/취소 버튼과 혼동될 가능성이 낮습니다.
- 파일 상태 조회가 “주문 상품의 업로드 파일 확인” 기능이라는 맥락이 분명합니다.

### 2순위

주문 상품 정보 영역과 결제 정보 영역 사이에 넣을 수 있습니다.

이 경우에도 안내 박스가 결제 금액, 배송 상태, 취소/교환/반품 버튼과 섞이지 않도록 여백을 충분히 둡니다.

## 4. 피해야 할 위치

아래 위치에는 1차 적용하지 않습니다.

- 상품상세 화면: 이 화면은 파일 업로드 위젯만 두는 위치입니다.
- 장바구니 화면: 주문 완료 전 단계라 파일 상태조회 안내 위치로 적합하지 않습니다.
- 주문조회 목록의 각 주문 행 내부: 반복 안내 박스가 많아져 화면이 복잡해질 수 있습니다.
- 결제 금액 영역 안쪽
- 배송 상태 버튼 주변
- 취소/교환/반품 버튼 바로 옆
- 관리자 전용 메모 영역
- 모바일에서 표 너비를 밀어낼 수 있는 좁은 테이블 내부

## 5. 스니펫 버전 선택 기준

스니펫 초안은 `docs/CAFE24_FILE_STATUS_SNIPPET.md` 문서를 기준으로 사용합니다.

1차 적용에서는 주문상세 화면이 이미 정보가 많은 화면이라는 점을 고려해 **짧은 문구 버전**을 우선 검토하는 것을 권장합니다.

권장 버튼 문구:

```text
업로드 파일 상태 조회하기
```

권장 안내 방향:

- 업로드한 인쇄용 파일의 상태를 조회할 수 있다고 안내합니다.
- 조회에는 Cafe24 주문번호와 업로드 파일 ID가 필요하다고 안내합니다.
- 품목별 주문번호 끝에 `-01`, `-02`가 붙어 있어도 조회 가능하다고 안내합니다.
- 상태 조회 화면에서는 파일 다운로드를 제공하지 않는다고 안내합니다.
- 재업로드가 필요한 경우 안내받은 재업로드 링크를 이용하라고 안내합니다.

## 6. Cafe24에서 확인할 템플릿 후보

실제 Cafe24 스킨마다 파일명과 구조가 다를 수 있으므로, 적용 전 아래 영역을 확인합니다.

- 마이페이지 주문조회 목록 템플릿
- 마이페이지 주문상세 템플릿
- 주문상세 화면의 주문 상품 정보 HTML
- 업로드 파일 ID가 상품 옵션으로 표시되는 영역
- PC 스킨과 모바일 스킨 분리 여부
- 모바일 주문상세 별도 템플릿 여부
- 스킨 공통 CSS가 안내 박스 스타일을 덮어쓰는지 여부

Cafe24 관리자에서 확인할 수 있는 경로는 스킨 상태에 따라 다를 수 있습니다.

예상 확인 경로:

```text
Cafe24 관리자 > 디자인 > 디자인 보관함 또는 스마트디자인 편집 > 사용 중인 스킨 > 마이페이지 / 주문조회 / 주문상세 관련 파일
```

정확한 파일명은 운영 중인 Cafe24 스킨에서 직접 확인한 뒤 확정합니다.

## 7. 실제 적용 체크리스트

운영 스킨에 적용하기 전 아래 항목을 확인합니다.

- [ ] 주문상세 화면에서 주문번호가 고객에게 보인다.
- [ ] 주문상세 화면의 주문 상품 정보 또는 상품 옵션 영역에서 업로드 파일 ID가 고객에게 보인다.
- [ ] 안내 박스가 주문 상품 정보 영역 아래 또는 업로드 파일 ID가 보이는 영역 근처에 들어간다.
- [ ] 버튼이 `https://perpackage-cafe24-file-upload-app.vercel.app/file-status`로 이동한다.
- [ ] `/file-status`에서 기본 주문번호로 조회할 수 있다.
- [ ] `/file-status`에서 품목별 주문번호 끝의 `-01`, `-02`가 붙은 값도 조회할 수 있다.
- [ ] 안내 박스에 고객 다운로드 버튼이 없다.
- [ ] 안내 박스에 `storage_path`, signed URL, token, Webhook raw payload가 노출되지 않는다.
- [ ] 상품상세 화면에는 기존 파일 업로드 위젯만 남아 있다.
- [ ] 장바구니 화면에는 파일 상태조회 안내 박스를 넣지 않았다.
- [ ] PC 화면에서 기존 주문상세 레이아웃이 깨지지 않는다.
- [ ] 모바일 화면에서 안내 박스가 화면 밖으로 밀리지 않는다.
- [ ] 취소, 교환, 반품, 배송 상태 버튼과 혼동되지 않는다.

## 8. 1차 적용 범위

1차 적용 범위는 다음으로 제한합니다.

- Cafe24 고객 마이페이지 주문조회 > 주문상세 화면에 작은 안내 박스 추가
- `/file-status`로 이동하는 버튼 추가
- 고객이 주문번호와 업로드 파일 ID를 직접 입력하는 방식 유지
- URL 자동 입력 없음
- Cafe24 화면에서 `file_id` 자동 파싱 JS 없음
- 고객 다운로드 버튼 없음
- 재업로드 링크 자동 표시 없음

## 9. 1차에서 제외할 것

아래 항목은 1차 적용에서 제외합니다.

- 주문조회 목록의 각 주문별 상태조회 버튼 자동 생성
- `/file-status?order_id=...&file_id=...` 자동 입력 링크
- Cafe24 주문상세 HTML에서 업로드 파일 ID 자동 파싱
- 고객용 파일 다운로드 기능
- 재업로드 링크 자동 노출
- SMS, 카카오톡, 이메일 자동 발송
- Cafe24 스마트디자인 운영 스킨의 복잡한 JS 수정

## 10. 2차 개선 후보

1차 적용 후 고객 사용성과 운영 문의량을 보고 아래 항목을 검토할 수 있습니다.

- 마이페이지 주문조회 목록 상단에 짧은 공통 안내문 추가
- 마이페이지 메인에 파일 상태조회 바로가기 카드 추가
- 주문상세 화면에서 주문번호와 업로드 파일 ID를 자동 감지해 `/file-status`에 전달
- `/file-status?order_id=...&file_id=...` 자동 입력 지원
- 모바일 주문상세 전용 짧은 문구 버전 적용
- 조회 시도 제한 또는 rate limit 강화
- 고객 안내 문자/카카오톡 문구에 `/file-status` 링크 추가

## 11. 보안 기준

아래 기준을 지킵니다.

- 1차에서는 URL에 주문번호와 업로드 파일 ID를 자동으로 붙이지 않습니다.
- `file_id` 단독 조회 방식은 사용하지 않습니다.
- 고객 화면에는 파일 다운로드 버튼을 제공하지 않습니다.
- 고객 화면에는 내부 저장 경로를 표시하지 않습니다.
- 고객 화면에는 signed URL 원문을 표시하지 않습니다.
- 고객 화면에는 raw token 또는 `token_hash`를 표시하지 않습니다.
- 고객 화면에는 Webhook raw payload 전체를 표시하지 않습니다.
- 고객 화면에는 Cafe24 Admin API 응답 전체를 표시하지 않습니다.
- 재업로드가 필요한 경우 기존 `/reupload?token=...` 링크를 별도로 안내합니다.
- 재업로드 raw token은 DB에 저장하지 않는 현재 원칙을 유지합니다.
