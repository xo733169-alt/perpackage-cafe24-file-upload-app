# Cafe24 마이쇼핑 메인 메뉴 사용 여부 점검 문서

- 작성 기준: 2026-07-06
- 대상 화면: Cafe24 고객 마이쇼핑 메인
- 문서 성격: UI 개선 전 메뉴/기능 점검 문서

## 1. 문서 목적

이 문서는 Cafe24 마이쇼핑 메인 UI를 개선하기 전에 현재 메뉴와 기능을 먼저 분류하기 위한 점검 문서입니다.

마이쇼핑 메인은 주문조회, 회원정보, 쿠폰, 적립금, 배송주소록, 게시물, 문의 기능과 연결되어 있습니다. 따라서 디자인이 불편하거나 메뉴가 많아 보여도 바로 삭제하지 않고, 실제 사용 여부와 작동 여부를 먼저 확인해야 합니다.

이번 문서의 목적은 다음과 같습니다.

- 마이쇼핑 메인 UI를 개선하기 전 현재 메뉴를 분류합니다.
- 실제 사용하는 메뉴와 숨김 후보 메뉴를 구분합니다.
- 주문, 회원, 배송, 쿠폰, 적립금 등 고객 기능이 깨지지 않도록 합니다.
- 삭제보다 숨김 또는 위치 조정 중심으로 접근합니다.
- 실제 개선 전 PC와 모바일에서 작동 여부를 확인합니다.

| 사용 여부 확인 필요 | 실제 고객이 쓰는지, 운영자가 쓰는지 먼저 확인해야 하는 메뉴 | 관심상품, 최근본 상품, 좋아요, 게시물관리 | 클릭 테스트와 운영 사용 여부 확인 후 유지/숨김 결정 |

| 삭제 금지, 숨김 전 확인 필요 | 고객 기능과 연결되어 있어 삭제하면 문제가 생길 수 있는 메뉴 | 주문조회, 회원정보, 배송주소록, 취소/반품/교환 관련 메뉴 | 삭제하지 않고, 숨김도 실제 기능 확인 후 판단 |

## 5. 파일 상태조회 바로가기 추가 검토

현재 파일 상태조회 1차 핵심 위치는 고객 마이페이지 주문조회 > 주문상세 화면입니다.

마이쇼핑 메인에는 필수는 아니지만, 보조 바로가기 카드로 “업로드 파일 상태 조회”를 추가할 수 있습니다.

| 구분 | 내용 |
|---|---|
| 장점 | 고객이 파일 상태조회 페이지를 더 쉽게 찾을 수 있음 |
| 단점 | 마이쇼핑 메인에서는 주문번호와 업로드 파일 ID가 바로 보이지 않아 고객이 다시 주문상세를 확인해야 할 수 있음 |
| 위험 | 메뉴가 더 많아져 마이쇼핑 메인이 더 복잡해질 수 있음 |
| 추천 | 1차에서는 주문상세 안내 박스를 유지하고, 마이쇼핑 메인 바로가기는 2차 후보로 둠 |

추가를 검토한다면 버튼 문구는 “업로드 파일 상태 조회”처럼 짧게 두고, 안내문에는 주문번호와 업로드 파일 ID가 필요하다고 명확히 적어야 합니다.

## 6. 마이쇼핑 UI 개선 방향 초안

- 핵심 메뉴 중심으로 재정렬합니다.
- 주문조회, 파일 상태조회, 상담문의, 배송주소록을 우선 노출합니다.
- 사용 빈도가 낮은 메뉴는 하단 또는 접힘 영역으로 이동합니다.
- 작동하지 않는 메뉴는 바로 삭제하지 않고 숨김 후보로 관리합니다.
- 고객센터 영역은 네이버톡톡, 카카오톡채널, 폼메일 등 실제 사용하는 채널 중심으로 정리합니다.
- PC와 모바일에서 동일하게 이해되는 문구를 사용합니다.
- 주문, 계정, 배송, 취소/반품/교환 기능은 기존 기능이 깨지지 않도록 위치 변경도 단계적으로 진행합니다.
