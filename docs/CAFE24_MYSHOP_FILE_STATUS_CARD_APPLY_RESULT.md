# Cafe24 마이쇼핑 파일 상태조회 바로가기 카드 적용 완료 기록

- 작성 기준: 2026-07-06
- 대상 화면: Cafe24 고객 마이쇼핑 메인
- 문서 성격: 운영 반영 완료 기록

## 1. 적용 목적

Cafe24 고객 마이쇼핑 메인에서 고객이 업로드 파일 상태 조회 페이지를 쉽게 찾을 수 있도록 `업로드 파일 상태 조회` 바로가기 카드를 추가했습니다.

운영 방향은 다음과 같습니다.

- 주문상세 화면의 안내 박스는 파일 상태 조회의 핵심 안내 위치로 유지합니다.
- 마이쇼핑 메인 카드는 고객이 상태 조회 페이지를 다시 찾기 쉽게 하는 보조 진입점입니다.
- 고객은 `/file-status`에서 주문번호와 업로드 파일 ID를 직접 입력합니다.
- 고객 화면에서는 파일 다운로드를 제공하지 않습니다.
- URL에 `order_id` 또는 `file_id`를 자동으로 붙이지 않습니다.

## 2. 실제 적용 위치

적용 위치는 다음과 같습니다.

- Cafe24 고객 마이쇼핑 메인
- `myshop_main` 카드 메뉴 영역
- `주문조회` 카드 바로 다음
- `회원정보` 카드 앞

기존 메뉴 처리 기준은 다음과 같습니다.

- 기존 `주문조회` 메뉴 유지
- 기존 `회원정보` 메뉴 유지
- 기존 `배송 주소록 관리` 메뉴 유지
- 기존 메뉴 숨김 처리 없음
- 기존 `module`, `{$변수}`, `onclick` 구조 변경 없음

## 3. 실제 적용 방식

적용된 카드 정보는 다음과 같습니다.

| 항목 | 내용 |
|---|---|
| 카드명 | 업로드 파일 상태 조회 |
| 보조 문구 | 주문번호와 업로드 파일 ID가 필요합니다. |
| 링크 | `https://perpackage-cafe24-file-upload-app.vercel.app/file-status` |
| 열림 방식 | 새 탭 또는 새 창 |
| URL 자동 입력 | 사용하지 않음 |
| file_id 자동 파싱 JS | 사용하지 않음 |
| 고객 입력 방식 | 고객이 주문번호와 업로드 파일 ID 직접 입력 |

1차 적용에서는 URL 파라미터 자동 입력이나 Cafe24 화면 내 `file_id` 자동 파싱을 넣지 않았습니다. 고객이 `/file-status` 화면에서 직접 값을 입력하는 방식입니다.

## 6. 남은 개선 후보

현재 적용은 기능상 문제 없이 확인되었습니다. 다만 추후 아래 항목은 별도 개선 후보로 남깁니다.

- `/file-status` 모바일 화면에서 긴 UUID 입력값이 일부 좁게 보이는 문제 개선
- 입력칸의 긴 업로드 파일 ID 표시 방식 개선
- 마이쇼핑 전체 UI 정리는 별도 작업으로 진행
- 좌측 FAQ 또는 고객센터 영역 숨김 후보는 별도 검토
- URL 자동 입력 방식은 2차 개선 후보로 유지
- 상태 조회 API rate limit 적용 검토
